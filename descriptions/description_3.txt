Detect suspicious BGP events where prefixes associated with critical DNS infrastructure are originated by an unexpected ASN and traffic is redirected to infrastructure outside the normal routing baseline. Build a baseline of expected origin ASNs and common path patterns for these prefixes, then flag sudden origin changes or path changes that redirect traffic toward a new network while preserving enough reachability for DNS responses to continue. Increase confidence when the event is visible across multiple regions or collectors and when the affected infrastructure serves a narrow or unusual subset of DNS queries during the anomaly window. Suppress alerts for approved provider migration and expected anycast reconfiguration.
